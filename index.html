User Details

Choose a unique username (3-64 characters). Avoid using email addresses as usernames for security reasons.
Optional friendly name for the user. This will be shown in the console and can be different from the username.

Select the user's department for organizational purposes and default permission assignments.

Access Type

Choose how the user will access AWS services. Programmatic access is for API/CLI usage, while AWS Management Console access provides web-based access.

Permissions

Assign permissions using groups, policies, or both. Groups are recommended for easier management. Follow the principle of least privilege.
Groups are easier to manage and follow AWS best practices. Direct policy attachment gives more granular control but requires more maintenance.
Available Groups
Select groups to add the user to. Groups contain policies that define permissions. Users inherit all permissions from their groups.

Review & Summary
